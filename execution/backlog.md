# Backlog

All tasks that are planned but not yet scheduled into a sprint.

> Rules: Append only. Never delete items — use Status field to track state. Each task must include priority, effort, and owner.

## Format

```
---
Date Added: YYYY-MM-DD
Task: [description]
Priority: HIGH | MEDIUM | LOW
Effort: [hours or story points]
Owner: [human or agent role]
Dependencies: [none or list]
Status: BACKLOG | IN_SPRINT | DONE | CANCELLED
Notes: [any context]
---
```
